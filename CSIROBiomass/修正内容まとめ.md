# CSIRO Biomass コンペティション 修正内容まとめ

## 問題の背景

### 初期状態
- **CVスコア（Weighted R2）**: 0.7403
- **パブリックスコア**: 0.48
- **問題**: CVスコアとパブリックスコアの間に大きな乖離があり、過学習が疑われる状態

### 過学習の原因分析

1. **PCA次元数が多すぎる可能性**
   - 固定で512次元を使用
   - 訓練サンプル数に対して次元数が多すぎる場合、過学習を引き起こす可能性

2. **正則化パラメータが不十分**
   - 学習率、サブサンプリング、L1/L2正則化などのパラメータが過学習を防ぐには弱い可能性

3. **早期停止の未実装**
   - 当初は早期停止を実装しようとしたが、GridSearchCVとの互換性の問題で実装できず

## 実施した修正内容

### 1. PCA次元数の最適化

#### 修正前
```python
target_components = 512  # 目標次元数
n_components = min(target_components, n_samples - 1)  # サンプル数-1まで（安全のため）
```

#### 修正後
```python
# 説明分散比の目標（95%）で次元数を決定
pca_for_selection = PCA(random_state=42)
pca_for_selection.fit(X_train_scaled)

cumsum_var = np.cumsum(pca_for_selection.explained_variance_ratio_)
target_variance = 0.95  # 95%の分散を説明
n_components = np.argmax(cumsum_var >= target_variance) + 1
# ただし、訓練サンプル数と256のうち小さい方に制限（過学習防止）
n_components = min(n_components, n_samples - 1, 256)  # 最大256次元に制限
n_components = max(n_components, 50)  # 最小50次元は保持
```

#### 変更内容
- **説明分散比95%を達成する最小次元数を自動選択**
- **最大次元数を256に制限**（過学習防止）
- **最小次元数を50に設定**（情報損失を防ぐ）

#### 効果
- 不要な特徴量を削減して過学習を抑制
- データの特性に応じて適切な次元数を自動選択

### 3. 早期停止（Early Stopping）の実装試行と修正

#### 試行した実装
```python
base_lgbm = MultiOutputRegressor(
    LGBMRegressor(
        verbosity=-1,
        early_stopping_rounds=50,  # 早期停止を追加
        force_row_wise=True
    )
)
```

#### 問題発生
```
ValueError: For early stopping, at least one dataset and eval metric is required for evaluation
```

#### 原因
- `GridSearchCV`と`MultiOutputRegressor`の組み合わせでは、`early_stopping_rounds`を使用するために`eval_set`と`eval_metric`が必要
- しかし、`GridSearchCV`のクロスバリデーションループ内では、これらのパラメータを直接提供できない

#### 修正後の実装
```python
base_lgbm = MultiOutputRegressor(
    LGBMRegressor(
        verbosity=-1
    )
)
grid_search = GridSearchCV(
    base_lgbm,
    param_grid,
    cv=KFold(n_splits=5, shuffle=True, random_state=42),
    scoring=weighted_scorer,
    n_jobs=-1,
    verbose=1,
    error_score='raise'  # エラーの詳細を確認
)
```

#### 代替アプローチ
- `early_stopping_rounds`を削除
- `n_estimators`を適切な範囲（[100, 150]）に設定
- `GridSearchCV`のクロスバリデーションを通じて最適なイテレーション数を選択

#### 効果
- エラーを回避
- `GridSearchCV`がクロスバリデーションで最適な`n_estimators`を選択することで、過学習を抑制
