このドキュメントは、`agent-v1.py` の現在の挙動・前提条件・段階的な更新内容を記録するためのものです。

## 目的

- 現在の戦略を一元管理し、参照先を明確にする。

- 実験時の反復的な調整を容易にする。

## 現在の戦略

`agent-v1.py` は、最寄り目標を優先する拡張戦略に加えて、公転予測に基づく簡易偏差撃を実装しています。

- 観測データから惑星情報を取得する。
- 惑星を以下に分類する。
  - 自勢力の惑星（`owner == player`）
  - 非保有の惑星（`owner != player`）
- 各自勢力惑星について、ユークリッド距離が最小の非保有惑星を選択する。
- 各出撃元には最低限の守備艦数を残し、全戦力の出し切りを防ぐ。
- 現在距離と推定艦速から到達ターンを近似計算する。
- 到着までに目標惑星が増産する守備艦数を見積もり、必要最小限の艦数を算出する。
- すでに同じ目標へ向かい、こちらより先に到着する自軍艦隊を差し引いてオーバーキルを防ぐ。
- `angular_velocity` と `initial_planets` を用いて到着時の目標位置を予測する。
- 出撃元から予測目標位置への直線経路が太陽を横切る場合、その射出をキャンセルする。
- 出撃可能な艦数を満たす場合のみ、予測座標に向けて艦隊を発進する。

## 意思決定フロー

1. `obs` から `player` と `planets` を読み取る。
2. 生データの惑星配列を `Planet` オブジェクトへ変換する。
3. `my_planets` と `targets` を構築する。
4. 各出撃元惑星について以下を実行する。
   - 直接距離で最寄りターゲットを探索する。
   - 現在距離と推定艦速から到達ターンを算出する。
   - 相手所有惑星のみ増産を加味して、到着時の守備艦数を見積もる。
   - 同じ目標に先着する自軍艦隊数を見積もる。
   - 必要艦数を `predicted_garrison + 1 - friendly_inbound_ships` で算出する。
   - 発射後に防衛マージンを残せるかを判定する。
   - 到達ターン後の目標座標を予測し、その方向へ発射角を計算する。
   - 予測経路が太陽半径内を通る場合、その射出を中止する。
   - 出撃元が必要艦数以上を保有している場合、艦隊を送る。
5. 移動命令を `[from_planet_id, angle, num_ships]` の形式で返す。

## パラメータと前提

- **防衛マージン**: 各惑星には最低 `10` 隻を残す。値は定数として調整可能。
- **占領必要条件**: 到着時点での予測守備艦数 + 1 を送る。
- **生産考慮**: 相手所有惑星には `production * eta_turns` を加算する。中立惑星は増産しない前提で扱う。
- **艦隊トラッキング**: `fleets` を参照し、同じ目標へ向かう自軍艦隊のうち、こちらの新規艦隊より先着するものだけを差し引く。
- **距離評価**: 目標選定は意思決定時点の直線距離を用いる。
- **到達ターン推定**: 距離/推定艦速で1回近似する。
- **公転予測**: `initial_planets` を基準に `theta = theta0 + omega * (step + t)` で将来角度を算出する。
- **適用範囲**: 公転条件を満たす惑星のみ将来位置予測を適用し、静止惑星は現在座標を使う。
- **太陽回避判定**: 出撃元から予測目標位置までの線分と太陽円（中心 `(50, 50)`、半径 `10`）の交差で判定する。

## 既知の制約

- 生産力や戦略的価値に基づく目標優先度がない。
- 敵艦隊の脅威推定を行っていない。
- 到着前に他艦隊が目標へ与える増援・削減は見込んでいない。
- 飛行中艦隊の目標推定は現在の惑星配置に対する直線衝突推定であり、公転中の厳密な着弾先までは追っていない。
- 到達ターン推定は単回近似であり、反復補正は行っていない。
- 太陽回避は単純な発射可否判定のみであり、迂回ルートの探索は行っていない。
- 防衛マージンは固定値であり、惑星価値や敵脅威に応じた動的調整は行っていない。
- 同一ターン内の複数出撃に対する中長期連携がない。

## 更新ログ

変更ごとに1エントリを追加し、新しいものを上に記載します。

### 2026-04-22 - 艦隊トラッキングによるオーバーキル防止を追加

- `fleets` 観測を使って、同じ目標へ先着する自軍艦隊の艦数を見積もるよう更新した。
- 追加射出数を「到着時必要艦数」から「先着する自軍艦隊」を差し引いた不足分だけに制限した。
- 不足分が 0 以下のときは新規発射しない方針を追加した。

### 2026-04-22 - 到着時生産を考慮した射出数計算を追加

- 相手所有惑星について、到着までの生産量を加味した将来守備艦数を見積もるよう更新した。
- 必要艦数を「現在守備」ではなく「到着時予測守備 + 1」で計算するよう変更した。
- 中立惑星はゲーム仕様に合わせて増産なしとして扱う方針を明記した。

### 2026-04-22 - 防衛マージンを追加

- 各惑星からの出撃時に最低 `10` 隻を残す制約を導入した。
- 全戦力を送り出して直後に奪われるリスクを抑えるよう、発射条件を強化した。
- 防衛マージンは定数化し、今後の調整をしやすくした。

### 2026-04-22 - 太陽衝突回避フィルタを追加

- 出撃元から予測目標位置への線分が太陽円を横切るかを判定するヘルパーを追加した。
- 太陽へ衝突する見込みの射出はキャンセルし、無駄な艦隊損失を防ぐよう更新した。
- 迂回は行わず、現時点では安全な直線射出のみを許可する方針とした。

### 2026-04-22 - 公転予測による偏差撃の基礎を追加

- `angular_velocity` と `initial_planets` を使う未来座標予測関数を追加した。
- 目標の現在位置ではなく、到達推定ターン後の予測位置に向けて発射角を計算するよう更新した。
- 到達ターンは距離と艦速からの単回近似で算出する方式を採用した。

### 2026-04-22 - 初版作成

- 最寄り目標戦略の基礎仕様を文書化した。
- 意思決定フロー、前提、制約の各セクションを追加した。
- 継続的な改善記録のための更新テンプレートを整備した。

## 次の改善案

- 生産力・距離・所有リスクを加味した目標スコアリングを導入する。
- 高価値惑星に対する複数出撃元の協調攻撃を実装する。
- ゲーム進行段階（序盤/中盤/終盤）に応じた方針切り替えを追加する。
- 到達ターン推定の反復補正（予測位置で再計算）を導入する。
- 太陽を避けるための代替目標選択や迂回戦略を導入する。
- 防衛マージンを固定値から動的な脅威ベースに拡張する。
- 目標への敵増援や途中戦闘も含めた将来戦力予測を導入する。
- 公転惑星に対する飛行中艦隊の着弾先推定を、現在位置ベースから将来位置ベースへ改善する。
