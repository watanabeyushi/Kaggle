# Agent v1 README

このドキュメントは、`agent-v1.py` の現在の挙動・前提条件・段階的な更新内容を記録するためのものです。

## 目的

- 現在の戦略を一元管理し、参照先を明確にする。
- 各変更の意図と背景を記録する。
- 実験時の反復的な調整を容易にする。

## 現在の戦略

`agent-v1.py` は、優先順位付きのスコア戦略を実装しています。行動優先順位は `インターセプト > ROI攻撃 > 補給` です。

- 観測データから惑星情報を取得する。
- 惑星を以下に分類する。
  - 自勢力の惑星（`owner == player`）
  - 非保有の惑星（`owner != player`）
- 敵同士の戦闘後で守備が薄くなるタイミングを監視し、先着インターセプト候補を最優先で狙う。
- 通常攻撃では、距離だけでなく `production / (ships_needed * eta)` 系の ROI 指標で目標を選ぶ。
- 攻撃候補が弱い後方惑星は、前線寄りの自惑星へ補給を行う。
- 各出撃元には最低限の守備艦数を残し、全戦力の出し切りを防ぐ。
- 惑星中心ではなく、実際の発射位置（惑星半径の外側）から照準を計算する。
- 到達時間は整数ターンではなく連続時間で近似し、未来位置と距離を反復更新して収束させる。
- 到着までに目標惑星が増産する守備艦数を見積もり、必要最小限の艦数を算出する。
- すでに同じ目標へ向かい、こちらより先に到着する自軍艦隊を差し引いてオーバーキルを防ぐ。
- `angular_velocity` と `initial_planets` を用いて連続時間で目標位置を予測する。
- moving target には通常命中に加え、惑星移動による sweep 命中も狙う。
- 出撃元中心ではなく実発射位置から太陽横断を判定し、その射出をキャンセルする。
- 出撃可能な艦数を満たす場合のみ、予測座標に向けて艦隊を発進する。

## 意思決定フロー

1. `obs` から `player` と `planets` を読み取る。
2. 生データの惑星配列を `Planet` / `Fleet` オブジェクトへ変換する。
3. 飛行中艦隊から目標惑星ごとの到着イベントを推定する。
4. 非保有惑星ごとに、敵同士の戦闘後を狙えるインターセプト窓を構築する。
5. 自惑星を前線/後方に簡易分類する。
6. 各自惑星について候補を評価する。
   - インターセプト候補を評価する。
   - ROI による通常攻撃候補を評価する。
   - 後方惑星なら補給候補を評価する。
   - 実発射位置を計算する。
   - 連続時間ベースで未来位置と到達時間を反復更新する。
   - moving target では sweep 命中候補も含めて照準点を決める。
7. 優先順位 `インターセプト > ROI攻撃 > 補給` で採用し、同一ターンの二重使用を避ける。
8. 移動命令を `[from_planet_id, angle, num_ships]` の形式で返す。

## パラメータと前提

- **防衛マージン**: 各惑星には最低 `10` 隻を残す。値は定数として調整可能。
- **反復エイム回数**: `3` 回。定数として調整可能。
- **発射点補正**: 艦隊は惑星中心ではなく、`planet.radius + 0.1` の外側から発射される前提で扱う。
- **占領必要条件**: 到着時点での予測守備艦数 + 1 を送る。
- **生産考慮**: 相手所有惑星には `production * eta_turns` を加算する。中立惑星は増産しない前提で扱う。
- **艦隊トラッキング**: `fleets` を参照し、同じ目標へ向かう自軍艦隊のうち、こちらの新規艦隊より先着するものだけを差し引く。
- **ROI 指標**: 通常攻撃の優先度は概ね `production / (ships_needed * eta_turns)` で評価する。
- **インターセプト条件**: 非自軍同士の占領変化が起きた次ターンを狙い、ちょうどそのターンに届く場合のみ採用する。
- **補給条件**: 前線寄り自惑星の不足戦力を、後方惑星の余剰艦で埋める。
- **前線分類**: 敵に近い自惑星を最大 `3` 個まで前線とみなし、それ以外を後方とみなす。
- **到達時間推定**: 距離/推定艦速で初期化し、未来位置と距離を連続時間で有限回反復して更新する。
- **公転予測**: `initial_planets` を基準に `theta = theta0 + omega * (step + t)` で連続時間の将来角度を算出する。
- **sweep 命中**: moving target では、到着ターンの移動線分上で最も近い点も照準候補に含める。
- **飛行中艦隊推定**: 飛行中艦隊の着弾先は、将来時刻の惑星位置に対して反復推定する。
- **適用範囲**: 公転条件を満たす惑星のみ将来位置予測を適用し、静止惑星は現在座標を使う。
- **太陽回避判定**: 実発射位置から照準点までの線分と太陽円（中心 `(50, 50)`、半径 `10`）の交差で判定する。

## 既知の制約

- 敵艦隊の脅威推定を行っていない。
- 到着前に他艦隊が目標へ与える増援・削減は簡易推定であり、完全な将来戦闘シミュレーションは行っていない。
- 飛行中艦隊の目標推定は将来位置ベースへ改善したが、依然として近似であり完全な解析解ではない。
- 到達時間推定は有限回の反復近似であり、連続時間の厳密解を求めているわけではない。
- インターセプト窓は敵同士の占領変化を簡易検出したもので、複雑な多者戦闘の最適解ではない。
- 補給ロジックの前線判定と必要量はヒューリスティックであり、マップ全体最適ではない。
- sweep 命中は簡易モデルであり、全ターンの厳密な動く円衝突を完全再現しているわけではない。
- 太陽回避は単純な発射可否判定のみであり、迂回ルートの探索は行っていない。
- 防衛マージンは固定値であり、惑星価値や敵脅威に応じた動的調整は行っていない。

- 不足分が 0 以下のときは新規発射しない方針を追加した。

- 必要艦数を「現在守備」ではなく「到着時予測守備 + 1」で計算するよう変更した。
- 中立惑星はゲーム仕様に合わせて増産なしとして扱う方針を明記した。

### 2026-04-22 - 防衛マージンを追加

- 各惑星からの出撃時に最低 `10` 隻を残す制約を導入した。
- 全戦力を送り出して直後に奪われるリスクを抑えるよう、発射条件を強化した。
- 防衛マージンは定数化し、今後の調整をしやすくした。

### 2026-04-22 - 太陽衝突回避フィルタを追加

- 出撃元から予測目標位置への線分が太陽円を横切るかを判定するヘルパーを追加した。
- 太陽へ衝突する見込みの射出はキャンセルし、無駄な艦隊損失を防ぐよう更新した。
- 迂回は行わず、現時点では安全な直線射出のみを許可する方針とした。
