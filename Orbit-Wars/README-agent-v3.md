# Agent v3 README

このドキュメントは、`agent-v3.py` の現在の挙動・前提条件・段階的な更新内容を記録するためのものです。

## 目的

- `implementation_log.pdf` で整理された戦略を、再現可能な形でコードへ落とし込む。
- `agent-v1.py` の高精度命中ロジックを維持しつつ、意思決定をより戦略寄りに拡張する。
- 今後の実験で、どの評価項目や待機判断が効いたかを追跡しやすくする。

## 現在の戦略

`agent-v3.py` は、`v1` の命中精度改善を土台にした優先順位付きのミッション戦略を実装しています。行動優先順位は `救援 > 再奪取 > 価値ベース占領` です。

- 観測データから惑星・艦隊・初期惑星配置・角速度を取得する。
- `v1` 同様、惑星中心ではなく実際の発射位置から未来位置へ向けて偏差撃ちを行う。
- moving target に対して、連続時間の反復エイムと sweep 命中検証を使って角度を決める。
- 太陽を横切る射出はキャンセルし、命中検証を通った解のみ採用する。
- 飛行中艦隊の到着先を未来位置ベースで推定し、各惑星の到着イベント台帳を構築する。
- 到着イベント台帳から、各惑星の将来 owner / garrison を簡易シミュレーションする。
- 防衛では、近い将来に危険な自惑星へ救援を送る。
- 再奪取では、近い将来に失う見込みの自惑星に対して、失陥直後の取り返し候補を評価する。
- 占領では、単純な最寄りではなく `target_value` に基づいて目標を選ぶ。
- `target_value` では以下を考慮する。
  - 生産量と到着後の将来収益
  - 序盤ほど重く見るテンポ補正
  - 周辺クラスタ密度
  - 敵との競争距離
  - 中継惑星としての gateway bonus
- 出撃元を空にしすぎないよう、序盤・前線距離・自軍惑星数・生産量を使った `source optionality cost` を入れる。
- 競合しやすく敵前線に近い目標には `target stability cost` をかける。
- 艦隊速度の非線形性を考慮し、今すぐ出す案と、少し待ってより速い艦数で出す案を比較する。
- 待ったほうが総到着ターンが早くなる場合は、そのターンは発射を見送る。
- 同一ターンの採用済み移動は `planned_arrivals_by_target` に積み、後続候補での過剰投入を抑える。

## 意思決定フロー

1. `obs` から `player`、`planets`、`fleets`、`initial_planets`、`angular_velocity` を取得する。
2. 惑星・艦隊を内部表現へ変換する。
3. `v1` ベースの未来位置推定と命中検証で、既存飛行中艦隊の着弾先と ETA を推定する。
4. 惑星ごとの到着イベント台帳を作る。
5. 自惑星ごとに、近い将来の失陥や危険状態を簡易シミュレーションする。
6. 各出撃元について、以下の順に候補を評価する。
   - 救援候補
   - 再奪取候補
   - 非保有惑星の占領候補
7. 各候補では、必要艦数の下限を求めた上で `preferred launch` を評価する。
8. `preferred launch` では以下を比較する。
   - 今すぐ最小艦数で出す案
   - ETA を短縮できる艦数しきい値案
   - 数ターン待機して、より高速化してから出す案
9. 待機のほうが総到着ターンで有利なら、その候補は今ターン不採用にする。
10. 採用候補のスコアが閾値を超えた場合のみ `[from_planet_id, angle, num_ships]` を返す。

## パラメータと前提

- **防衛マージン**: 固定値ではなく、`BASE_DEFENSE_MARGIN=8` を基準に序盤・前線距離・惑星数・生産量で動的補正する。
- **反復エイム回数**: `3` 回。`v1` と同様に有限回で未来位置と ETA を収束させる。
- **発射点補正**: `planet.radius + 0.1` の外側から発射する前提で扱う。
- **命中検証**: `strict / relaxed` を候補コストとミッション種別から切り替える。
- **到達先推定**: 飛行中艦隊も、現在位置ではなく予測未来位置ベースで着弾先を推定する。
- **未来位置予測**: 公転惑星は `initial_planets` と `angular_velocity` を使って `theta = theta0 + omega * (step + t)` で予測する。
- **sweep 命中**: moving target の到着ターン前後の移動線分も命中候補に含めるが、検証が通った場合のみ採用する。
- **速度最適化**: ETA を短縮する艦数候補を逆算し、現在の最小必要艦数案と比較する。
- **待機探索**: 最大 `3` ターン先まで待機を比較し、待機後の総到着ターンが短ければ今ターンは待つ。
- **将来戦闘推定**: 現在飛行中の艦隊と当ターン採用済み自軍到着だけを使った簡易シミュレーションで owner / garrison を予測する。
- **価値評価**: `target_value` は生産量、テンポ、クラスタ密度、競争距離、中継価値を合成したヒューリスティック。
- **柔軟性コスト**: 序盤ほど、前線ほど、自軍惑星数が少ないほど、また高生産ソースほど「残す価値」を重くする。
- **安定性コスト**: 敵に先着されやすい目標、敵前線に近い目標、敵到着予定が近い目標を減点する。
- **ミッション優先順位**: `救援 > 再奪取 > 価値ベース占領` を基本とする。

## 既知の制約

- 救援と再奪取は簡易シミュレーションであり、複数プレイヤーの全分岐を厳密には展開していない。
- 待機判断では、待機中に出撃元が攻撃されるリスクや、待機中の味方増援までは十分に織り込めていない。
- `gateway bonus` は未来時点のサンプリング評価であり、多段最適化の厳密解ではない。
- 未来位置予測は公転惑星を対象にしており、彗星の将来軌道までは積極的に使っていない。
- 競争距離は主に現在位置ベースの近似であり、敵の速度最適化や待機戦略までは推定していない。
- source optionality / target stability の重みはヒューリスティックであり、盤面や人数設定ごとの最適保証はない。
- 同一ターンの複数ソース協調攻撃は、順次採用と予定到着の反映に留まっており、同時最適化ではない。
- 占領候補は高精度に命中させるが、途中で別惑星に当たる全ケースまでは完全には再現していない。
- 価値スコアのしきい値は経験的であり、マップ分布によっては強すぎる待機や弱すぎる発射が残る可能性がある。

## 更新ログ

変更ごとに1エントリを追加し、新しいものを上に記載します。

### 2026-04-30 - v3 初版作成

- `implementation_log.pdf` の要点を踏まえ、速度最適化付きの待機判断を追加した。
- `v1` の高精度命中ロジックを流用し、公転惑星への未来位置ベース偏差撃ちと sweep 命中検証を維持した。
- 救援、再奪取、価値ベース占領の 3 ミッション構成へ整理した。
- `target_value`、`source optionality cost`、`target stability cost`、`gateway bonus` を実装した。
- 既存飛行中艦隊から到着イベント台帳を作り、簡易な将来 owner / garrison 予測を入れた。

## 次の改善案

- 救援で「守り切れる最小艦数」を、損失ターン前後の複数ターンで再評価する。
- 再奪取と通常占領を、複数ソース同時最適化で比較できるようにする。
- `gateway bonus` を 2 段再帰からさらに深くし、序盤だけ重みを強める。
- 競争距離に敵の速度しきい値待機も織り込んで、競合評価を改善する。
- 彗星の未来軌道も価値評価へ組み込み、出現ターン周辺の意思決定を強化する。
- ミッション別に待機判断の保守性を変え、救援では即応性をより優先する。
