# MABe Mouse Behavior Detection - 1D CNN 実装履歴

## 概要

このドキュメントは、Kaggleコンペティション「MABe-mouse-behavior-detection」用の1D CNNモデルの実装履歴をまとめたものです。

**コンペティションURL**: https://www.kaggle.com/competitions/MABe-mouse-behavior-detection

## 実装の目的

マウスの行動検出タスクにおいて、時系列の追跡データ（体の部位の座標）から行動を分類する1D CNNモデルを実装しました。

### フェーズ4: 計算量削減のための最適化

#### 実装内容
データ量が多い場合の計算量削減のため、出現頻度の高い部位とアノテーションのみを使用する機能を実装しました。

#### 主な機能

1. **部位の出現頻度分析**
   ```python
   def analyze_body_parts_frequency(self, sample_size=100):
   ```
   - データセット全体を分析して部位の出現頻度を計算
   - サンプルデータを使用して効率的に分析
   - 上位N個の部位を選択、または閾値以上の部位を選択

2. **アノテーションの出現頻度分析**
   ```python
   def analyze_annotation_frequency(self):
   ```
   - アノテーションの出現頻度を計算
   - 最小出現頻度（デフォルト1%）以上のアノテーションのみを選択

3. **選択された部位のみを使用**
   - `load_tracking_data()`を修正
   - 選択された部位のみを特徴量として使用
   - 計算量とメモリ使用量を削減

4. **アノテーションフィルタリング**
   - `load_annotations()`を修正
   - 有効なアノテーションのみを使用
   - 低頻度のアノテーションを除外

#### パラメータ設定
```python
loader = MABeDataLoader(
    data_dir=DATA_DIR,
    top_body_parts=20,  # 上位20個の部位のみを使用
    min_annotation_freq=0.01  # 1%以上の出現頻度のアノテーションのみ
)
```

## モデルアーキテクチャ

### 1D CNN構造

```
入力: (batch_size, sequence_length=64, num_features)

↓ Permute (2, 1)
(batch_size, num_features, sequence_length)

↓ Conv1D (64 filters, kernel=3) + BatchNorm + MaxPool + Dropout
↓ Conv1D (128 filters, kernel=3) + BatchNorm + MaxPool + Dropout
↓ Conv1D (256 filters, kernel=3) + BatchNorm + MaxPool + Dropout
↓ Conv1D (512 filters, kernel=3) + BatchNorm + GlobalAveragePooling + Dropout

↓ Dense (256) + BatchNorm + Dropout
↓ Dense (128) + Dropout

出力: (batch_size, num_classes=15)
```

### ハイパーパラメータ

- **SEQUENCE_LENGTH**: 64フレーム
- **NUM_CLASSES**: 15クラス（MABeの行動数）
- **BATCH_SIZE**: 32
- **EPOCHS**: 100
- **LEARNING_RATE**: 0.001

## データ処理フロー

### 訓練データ処理

1. **データ読み込み**
   - train.csvから動画IDを取得
   - 各動画の追跡データ（train_tracking/）を読み込み
   - アノテーションデータ（train_annotation/）を読み込み

2. **前処理**
   - 部位の出現頻度分析（オプション）
   - アノテーションの出現頻度分析（オプション）
   - 選択された部位のみを使用
   - 特徴量の正規化（StandardScaler）
   - 時系列シーケンスの生成

3. **データ分割**
   - 訓練セット: 80%
   - 検証セット: 20%

### テストデータ処理

1. **データ読み込み**
   - test.csvからメタデータを読み込み
   - 各テスト行のフレーム範囲に対応する追跡データを抽出

2. **前処理**
   - 訓練データで学習したscalerを使用して正規化
   - フレーム数が不足する場合はパディング

3. **予測**
   - バッチ処理で予測を実行
   - 確率分布を取得

4. **提出ファイル生成**
   - 予測結果をsubmission.csv形式に変換
   - 検証と修正を実行
   - `/kaggle/working/submission.csv`に保存

## 主要なクラスと関数

### MABeDataLoaderクラス

データの読み込みと前処理を担当するクラス。

#### 主要メソッド

- `__init__(data_dir, top_body_parts, min_annotation_freq)`: 初期化
- `analyze_body_parts_frequency(sample_size)`: 部位の出現頻度分析
- `analyze_annotation_frequency()`: アノテーションの出現頻度分析
- `load_tracking_data(video_id)`: 追跡データの読み込み
- `load_annotations(video_id)`: アノテーションデータの読み込み
- `create_sequences(features, labels, sequence_length)`: 時系列シーケンスの生成
- `load_dataset(video_ids, sequence_length, analyze_frequency, sample_size)`: データセット全体の読み込み
- `load_test_data(sequence_length)`: テストデータの読み込み

### モデル構築関数

- `build_1dcnn_model(input_shape, num_classes)`: 1D CNNモデルの構築

### 訓練・評価関数

- `train_model(model, X_train, y_train, X_val, y_val, ...)`: モデルの訓練
- `evaluate_model(model, X_test, y_test)`: モデルの評価

### 予測・提出関数

- `predict_test_data(model, loader, sequence_length)`: テストデータの予測
- `create_submission(predictions, test_metadata, label_encoder, ...)`: 提出ファイルの生成
- `validate_submission(submission_df, sample_submission_path)`: 提出ファイルの検証

## 注意事項

1. **データ構造の確認**
